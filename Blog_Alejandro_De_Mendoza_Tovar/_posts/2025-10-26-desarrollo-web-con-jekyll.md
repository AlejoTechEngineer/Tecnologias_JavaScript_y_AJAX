**Jekyll** es una poderosa herramienta para crear sitios web estáticos de forma rápida, sencilla y eficiente. A través de este proyecto, aprendí a generar un blog personal utilizando **HTML**, **CSS** y **Markdown**, integrando imágenes, diseño responsivo y una estructura modular.

## ¿Qué es Jekyll?

Jekyll es un generador de sitios estáticos escrito en Ruby. A diferencia de los sistemas de gestión de contenido (CMS) tradicionales como WordPress, Jekyll no requiere bases de datos ni servidores complejos.

Esto lo hace ideal para blogs personales, portafolios y sitios de documentación técnica.

## Ventajas de usar Jekyll

### 1. Simplicidad

Todo el contenido se escribe en **Markdown**, un lenguaje de marcado ligero que facilita la redacción sin preocuparse por el HTML.

### 2. Velocidad

Al generar archivos estáticos, los sitios creados con Jekyll cargan rápidamente y requieren menos recursos del servidor.

### 3. Integración con GitHub Pages

Jekyll se integra perfectamente con GitHub Pages, permitiendo alojar sitios web de forma gratuita directamente desde un repositorio de GitHub.

### 4. Personalización total

Aunque Jekyll ofrece temas prediseñados, es completamente personalizable mediante HTML, CSS y Liquid (el lenguaje de plantillas).

## Mi Experiencia Creando este Blog

Al comenzar con Jekyll, me enfrenté a varios desafíos:

- **Configuración del entorno**: Instalar Ruby, Bundler y Jekyll en Windows requirió seguir varios pasos cuidadosamente.
- **Estructura de archivos**: Entender cómo funcionan las carpetas `_layouts`, `_includes` y `_posts` fue clave para organizar el proyecto.
- **Diseño responsivo**: Implementar CSS personalizado para que el blog se vea bien en dispositivos móviles y de escritorio.

## Lecciones Aprendidas

### HTML y CSS en acción

Aunque Jekyll automatiza gran parte del proceso, comprender HTML y CSS sigue siendo fundamental para personalizar el diseño.

### Markdown para contenido

Escribir en Markdown agiliza la creación de publicaciones sin perder el control sobre el formato.

### Control de versiones con Git

Trabajar con Jekyll me permitió practicar el uso de Git y GitHub, herramientas esenciales en el desarrollo moderno.

## Conclusión

Crear este blog con Jekyll fue una experiencia enriquecedora que me permitió aplicar conocimientos teóricos en un proyecto real. Ahora cuento con una plataforma personal donde puedo compartir mis ideas, proyectos y aprendizajes.

Si estás considerando crear tu propio blog, te recomiendo Jekyll como una excelente opción para iniciarte en el desarrollo web estático.
